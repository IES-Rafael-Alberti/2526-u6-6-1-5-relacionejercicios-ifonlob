En este caso la superclase es la clase `Animal` puesto que las subclases (`Perro` y `Gato`) que heredan de ésta comparten sus propiedades (variables de instancia) y métodos, además de agregar nuevos comportamientos y modificar los ya existentes como mencionaré más adelante.

**Segunda propuesta**:

En esta segunda propuesta la superclase es `Empleado` y las subclases son `EmpleadoRemoto` y `EmpleadoConExtras` puesto que éstas heredan comportamiento de la clase padre además de añadir nuevas funcionalidades.

II.c) ¿Existe otra forma de implementarlo que no sea herencia? (composición, interfaces, delegación, etc.)

RESPUESTA:

**Primera propuesta**:

Esta primera propuesta en vez de hacer uso de la herencia, cada una de las subclases podrían implementar una interfaz `HaceSonido` de la cual podrían implementar directamente a través de ella el método `hacerSonido` con sus respectivos modelos.

**Segunda propuesta**:

En esta segunda propuesta se podría llevar a cabo a través de composición en donde solo exista una clase `Empleado` que contenga todas las propiedades y todos los métodos tanto de la superclase como de las subclases,
pero que dependiendo de los valores de las propiedades específicas de las subclases se muestre un resultado u otro, como se puede apreciar a continuación:

```kotlin
class Empleado(
    val nombre: String,
    val apellido: String,
    val salarioBase: Double,
    val bonus: Double? = null,
    val horasTeletrabajo: Int? = null
) {
    fun trabajar() = println("$nombre $apellido está trabajando.")

    fun calcularSalario(): Double = salarioBase + (bonus ?: 0.0)

    fun solicitarVacaciones(dias: Int) {
        bonus?.let { println("$nombre solicita $dias vacaciones.") }
            ?: println("$nombre no tiene bonus/vacaciones.")
    }

    fun conectarVPN() {
        horasTeletrabajo?.let { println("$nombre conecta VPN ($it horas).") }
            ?: println("$nombre no es remoto.")
    }
}
```
